How to create a personal blog with Hugo

May 30, 2025   ·  3 min read

Hugo #

在本文中，我将详细介绍如何使用Hugo框架搭建个人博客，网络上不乏优秀教程，但我在首次搭建的时候仍然不免陷入误区，浪费大量时间在环境配置上，希望看到本篇博客的读者能够少走些弯路。

Hugo环境配置 #

  1. github中下载Hugo到本地

  2. 检查Hugo是否安装成功，显示出版本即可

   hugo -version
  1. 系统环境配置Hugo文件夹里的bin文件

Hugo创建博客站点 #

完成上述步骤后，可用在终端输入以下指令新建站点

hugo new site <name>

站点名称自定义，这将会在当前位置生成站点根目录。结构如下：

├── archetypes
│   └── default.md
├── hugo.toml         # 博客站点的配置文件
├── content             # 博客文章所在目录
├── data                
├── layouts             # 网站布局
├── static              # 一些静态内容
└── themes              # 博客主题

到此步，博客站点的创建便完成了，考虑到使用美观的Hugo主题，默认读者已经配置Git，如果没有配置的读者可以阅读其余教程完成Git配置后再接着阅读后续部分。

补充较为完整的根目录文件结构，如果只想简单快速构建自己的博客则可跳过阅读此部分。

├── content              # 存放所有内容文件
│   ├── _index.md        # 博客首页内容
│   ├── posts            # 博客文章
│   │   ├── first-post.md
│   │   └── second-post.md
│   └── pages            # 博客页面
│       └── about.md     # 关于页面
├── data                 # 存放站点数据
│   ├── authors.yml      # 作者信息
│   └── config.toml      # 站点配置文件
├── i18n                 # 国际化语言文件
│   ├── en.toml          # 英文语言文件
│   └── zh.toml          # 中文语言文件
├── layouts              # 存放页面模板
│   ├── _default         # 默认模板
│   ├── partials         # 模板片段
│   └── index.html       # 首页模板
├── assets               # 存放编译前的资源文件
│   ├── css              # 存放 CSS 源文件
│   ├── js               # 存放 JavaScript 源文件
│   └── images           # 存放图片源文件
├── resources            # 存放生成的资源文件
│   └── _gen             # 生成的资源文件
├── static               # 存放静态资源（如图片、CSS、JS）
│   ├── css              # 存放编译后的 CSS 文件
│   ├── js               # 存放编译后的 JavaScript 文件
│   └── images           # 存放图片文件
├── public               # 生成的网站文件
├── themes               # 存放主题文件
│   └── my-theme         # 自定义主题文件
│       ├── layouts      # 存放页面模板
│       ├── static       # 存放主题静态资源
│       └── theme.toml   # 主题配置文件
├── archetypes           # 内容模板文件
│   ├── default.md       # 默认内容模板
│   └── post.md          # 博客文章模板
└── hugo_build.lock         # Hugo 包管理文件

初始化git #

进入博客站点目录，在当前目录中初始化一个空的Git仓库。

cd quickstart
git init

配置主题 #

可进入官方主题仓库https://themes.gohugo.io/下载，若所下载主题支持子模块管理可使用下述指令直接添加为git子模块

git submodule add https://github.com/theNewDynamic/gohugo-theme-ananke.git themes/ananke

若所下载主题不支持，可手动下载压缩包到本地解压至站点根目录中themes文件夹中，并在hugo.toml中添加如下配置

theme = 'typo'

到这一步已经搭建好了博客框架，可在命令行启动Hugo开发服务器以查看网站

hugo server

博客内容创建 # 

hugo new content/posts/my-first-post.md

Hugo 会在 content/posts 目录中创建该文件

---
title: "我的第一篇文章"
date: 2022-11-20T09:03:20-08:00
draft: true #草稿内容
---

可添加一些内容到所创建的md文件中

---
title: "我的第一篇文章"
date: 2022-11-20T09:03:20-08:00
draft: true
---
## 简介

这是 **粗体** 文本，这是 *斜体* 文本。

访问 [Hugo](https://gohugo.io) 网站！

启动Hugo开发服务器，任选以下两个命令其中之一来包含文件草稿内容

hugo server --buildDrafts
hugo server -D

到这步已经基本完成了本地博客的构建，但是读者往往希望自己的博客网站可供他人浏览，因此下述部分将讲述如何结合Github page 及 Github Action来自动化发布博客内容。

配置网站 #

打开hugo.toml

baseURL = 'https://example.org/'
languageCode = 'en-us'
title = '我的新 Hugo 网站'
theme = 'ananke'

修改baseURL，可设置为自身的Github page网址

发布内容 #

使用Github Actions实现提交源码后自动发布

生成Actions secrets #

  1. 首先点击Github头像进入settings，划到最下方进入Developer Settings，点击Personal access tokens ，选择tokens （classic）

  2. 勾选其下的repo和workflows选项，选择Generate token生成，复制保存此token（只会显示一次）

  3. 假定已创建了Github page仓库，进入仓库Settings，选择Secrets and variable一栏，点击New repository secret，名称自取，将先前复制的token粘贴

配置actions文件 #

在根目录新建嵌套文件夹.github/workflows

在workflows文件夹中新建后缀为.yml的配置文件，名称自取，将以下配置写入：

name: github pages # 名字自取

on:
  push:
    branches:
      - main  # 这里的意思是当 main分支发生push的时候，运行下面的jobs，这里先改为github-actions

jobs:
  deploy: # 任务名自取
    runs-on: ubuntu-latest	# 在什么环境运行任务
    steps:
      - uses: actions/checkout@v2	# 引用actions/checkout这个action，与所在的github仓库同名
        with:
          submodules: true  # Fetch Hugo themes (true OR recursive) 获取submodule主题
          fetch-depth: 0    # Fetch all history for .GitInfo and .Lastmod

      - name: Setup Hugo	# 步骤名自取
        uses: peaceiris/actions-hugo@v2	# hugo官方提供的action，用于在任务环境中获取hugo
        with:
          hugo-version: 'latest'	# 获取最新版本的hugo
          # extended: true

      - name: Build
        run: hugo --minify	# 使用hugo构建静态网页

      - name: Deploy
        uses: peaceiris/actions-gh-pages@v3	# 一个自动发布github pages的action
        with:
          # github_token: ${{ secrets.GITHUB_TOKEN }} 该项适用于发布到源码相同repo的情况，不能用于发布到其他repo
          external_repository: krislinzhao/krislinzhao.github.io	# 发布到哪个repo
          personal_token: ${{ secrets.ACTION_ACCESS_TOKEN }}	# 发布到其他repo需要提供上面生成的personal access token
          publish_dir: ./public	# 注意这里指的是要发布哪个文件夹的内容，而不是指发布到目的仓库的什么位置，因为hugo默认生成静态网页到public文件夹，所以这里发布public文件夹里的内容
          publish_branch: main	# 发布到哪个branch

只需修改external_repository为自己的Github page仓库，personal_token: ${{ secrets.XXX }} 中的XXX改成生成Actions secrets时步骤3自取的secrets名称。

推送public文件夹到远程仓库 #

进入public文件夹

cd public

推送代码分支报错：可任选以下两种方式解决

  1. 初始化Git仓库，由于使用git init初始化Git仓库时默认分支名为master，为防止报错可将其修改为main再推送给远程仓库
git init -b main
  1. 远程关联Github page仓库后修改网站呈现页面为分支master

将其远程关联Github Pages仓库，使用仓库的ssh链接

git remote add origin <ssh>

推送本地文件到远程仓库

 git pull 
 git add .
 git commit -m "xxx"
 git push

推送完成后可在仓库Actions中查看运行时间，运行成功后可通过https://xxxx.github.io访问你的博客了。
